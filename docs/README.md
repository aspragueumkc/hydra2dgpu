# Documentation Index

This folder contains the project’s working documentation. The files are grouped by lifecycle rather than by a single flat list.

## How To Read This Folder

- `active/`: current plans, task boards, and session handoff notes.
- `reference/`: stable architecture notes, contracts, and implementation guides.
- `validation/`: checklists, acceptance notes, and test evidence.
- `archive/`: completed sprint cards, one-off completion notes, and superseded plans.
- `assets/`: PDFs, images, video, and other non-Markdown reference material.

## Top-Level Entry Point

The docs folder now keeps only this index at the top level.
All other content lives in lifecycle-specific subfolders.
